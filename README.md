# Repositories-Guidelines

Provides guidance on the current state of the github repos and creating, managing and approving repositories and making changes to them, and instructions/recommendations on licencing.

# Repository Requirements

- Repositories need a name. The name should be descriptive and follow the naming conventions of the AAGI organisation.
- Where possible, repositories need a license
- Repositories should have a README file that provides an overview of the project, how to install it, and how to use it.
- Repositories should have a CONTRIBUTING file that outlines how others can contribute to the project.
